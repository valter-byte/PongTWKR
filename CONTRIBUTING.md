---

## Project Structure and Branches

- **Until v1.0**  
  - The `/main` branch will host the CLI version of PongTWKR.  
  - Contributions here should remain stable, tested, and bug‑free.

- **After v1.0**  
  - The project will split into two branches:  
    - `/main`: Stable branch, focused on reliability, UI add‑ons, and well‑tested contributions.  
    - `/cli`: Experimental branch, open to cutting‑edge tweaks, new features, and community playground development.  

---

## Contribution Guidelines

1. **Code Quality**  
   - Write clean, readable, and well‑documented code.  
   - Follow existing coding style and conventions.  
   - Include comments where necessary to explain complex logic.

2. **Pull Requests**  
   - Clearly describe the purpose of your changes.  
   - Reference related issues if applicable.  
   - Ensure your code builds and runs without errors before submitting.

3. **Bug Reports**  
   - Please avoid vague reports such as *"uh, it broke my PC"*.  
   - Use a clear and structured format:
     ```
     ### Bug Report
     **Description:**  
     (Brief summary of the issue)

     **Steps to Reproduce:**  
     1. ...
     2. ...
     3. ...

     **Expected Behavior:**  
     (What should have happened)

     **Actual Behavior:**  
     (What actually happened)

     **Environment:**  
     - OS: (e.g., Ubuntu 22.04)
     - PongTWKR version: (e.g., v0.6)
     - Hardware details: (CPU, RAM, GPU if relevant)
     ```

4. **Feature Requests**  
   - Clearly explain the motivation and potential benefit.  
   - Provide examples or use cases.  
   - Keep requests realistic and aligned with project goals.

---

## Communication

- All issues, pull requests, and discussions must be in **English**.  
- Be respectful and constructive when interacting with other contributors.  
- Remember: `/cli` is the playground for experimentation, while `/main` is for stability.
